## Summary

Dictation output sometimes contains filler words ("um", "uh") and lacks paragraph breaks, even though the base cleanup prompt is supposed to always strip fillers and add paragraph spacing. The model appears to be returning raw transcription instead of cleaned text on some runs.

## Steps to reproduce

1. Launch Multimodal Voice Typer.
2. Dictate a passage containing filler words ("um", "uh") and multiple logical paragraphs.
3. Inspect the transcribed output.

## Expected behavior

The baseline cleanup prompt should always:
- Remove filler words (um, uh, er, etc.).
- Insert appropriate paragraph breaks.
- Apply basic punctuation and grammar fixes.

This is supposed to happen unconditionally regardless of format/tone overrides.

## Actual behavior

Some runs return output that still contains filler words and/or is delivered as a single run-on block with no paragraph separation — i.e. close to raw transcription rather than cleaned text.

### Root cause

The prompt sandwich guardrail added in f8eb671 put the full cleanup rules only in the `system` message and used a user-turn text part that told the model to transcribe the audio "verbatim (with the usual cleanup)". Gemini (and other multimodal audio models) weight the user turn containing the audio more heavily than the system message, and the word "verbatim" sitting directly next to the audio was biasing the model toward raw transcription — so filler words leaked through and paragraph breaks were dropped.

### Fix

In `app/src/transcription.py::OpenRouterClient._build_audio_payload`:

- Rewrote the pre-audio user-turn text to restate the non-negotiable cleanup requirements (strip fillers, insert blank-line paragraph breaks, punctuation/capitalization, output only cleaned text) right next to the audio, where multimodal models pay the most attention.
- Removed the word "verbatim" from the guardrail so it no longer competes with the cleanup instruction.
- Added a second short post-audio text part reiterating the two highest-risk rules (fillers + paragraph breaks) as a final reminder immediately before the model generates.

The system message (`CLEANUP_PROMPT`) is unchanged — it remains the authoritative spec. The user-turn text now reinforces it.
